## 공통 사항

- **Base URL**: `/api/v1`
- **인증**: JWT를 HttpOnly Cookie로 자동 전송 (클라이언트에서 별도 헤더 설정 불필요)
- **응답 형식**: JSON

**공통 응답 구조**:
```json
{
  "status": 200,
  "message": "성공",
  "data": { ... }
}
```

**공통 에러 코드**:

| HTTP 상태 코드 | 의미 | 설명 |
|---------------|------|------|
| 400 | Bad Request | 요청 형식 오류, 필수 파라미터 누락, 유효성 검증 실패 |
| 401 | Unauthorized | 인증 토큰 누락 또는 만료 |
| 403 | Forbidden | 해당 리소스에 대한 접근 권한 없음 |
| 404 | Not Found | 요청한 리소스가 존재하지 않음 |
| 409 | Conflict | 리소스 상태 충돌 |
| 410 | Gone | 요청한 리소스가 더 이상 존재하지 않음 (임시 파일 만료 등) |
| 429 | Too Many Requests | 동시 요청 과다 |
| 500 | Internal Server Error | 서버 내부 오류 |

## 인증 (Auth)

| Method | Endpoint | 설명 | 인증 |
|--------|----------|------|------|
| POST | `/auth/login` | 카카오 소셜 로그인 | X |
| POST | `/auth/refresh` | 토큰 재발급 | X (Refresh Token Cookie) |
| POST | `/auth/logout` | 로그아웃 | X (Refresh Token Cookie) |
| DELETE | `/auth/withdraw` | 회원 탈퇴 | O |

### POST /auth/login

카카오에서 받은 Access Token으로 로그인합니다. 첫 로그인 시 자동 회원가입됩니다. (FR-03)

- **Request Body**
```json
{
  "kakaoAccessToken": "카카오 Access Token"
}
```

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "로그인 성공",
  "data": {
    "user": {
      "id": 1,
      "nickname": "홍길동",
      "email": "hong@kakao.com",
      "profileImage": "https://..."
    }
  }
}
```

> JWT는 응답 body가 아닌 `Set-Cookie` 헤더로 전달됩니다.
> - `Set-Cookie: accessToken=...; HttpOnly; Secure; SameSite=Lax; Path=/; Max-Age=1800`
> - `Set-Cookie: refreshToken=...; HttpOnly; Secure; SameSite=Lax; Path=/api/v1/auth; Max-Age=1209600`

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 401 | 카카오 토큰 유효하지 않음 | "카카오 토큰이 유효하지 않습니다." |

### POST /auth/refresh

Refresh Token Cookie로 새로운 토큰 쌍을 발급합니다.

- **Request Body**: 없음 (Refresh Token은 Cookie에서 자동 전송)

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "토큰 재발급 성공"
}
```

> 새로운 Access Token, Refresh Token이 `Set-Cookie` 헤더로 전달됩니다.

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 401 | Refresh Token 만료/무효 | "Refresh Token이 만료되었습니다. 다시 로그인해주세요." |

### POST /auth/logout

Refresh Token을 무효화하고 쿠키를 삭제합니다. (FR-03)

- **Request Body**: 없음 (Refresh Token은 Cookie에서 자동 전송)

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "로그아웃 성공"
}
```

> 서버에서 `Set-Cookie`로 쿠키 만료 처리합니다. 로그아웃 요청 실패 시에도 클라이언트 측에서 쿠키를 삭제하여 로그아웃을 보장합니다.

### DELETE /auth/withdraw

회원 탈퇴. 사용자 데이터 및 관련 녹음/분석/질문 데이터가 모두 삭제됩니다. (FR-03)

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "회원 탈퇴 성공"
}
```

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 409 | 분석 진행 중 | "분석이 진행 중입니다. 완료 후 다시 시도해주세요." |

## 녹음 (Recording)

| Method | Endpoint | 설명 | 인증 |
|--------|----------|------|------|
| GET | `/recordings` | 녹음 목록 조회 | O |
| GET | `/recordings/{id}` | 녹음 상세 조회 | O |
| POST | `/recordings` | 녹음 등록 | O |
| PATCH | `/recordings/{id}` | 녹음 제목 수정 | O |
| DELETE | `/recordings/{id}` | 녹음 삭제 | O |

### GET /recordings

녹음 목록을 커서 기반으로 조회합니다. (FR-01)

- **Query Parameters**

| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| keyword | String | X | 제목 검색 |
| startDate | String | X | 조회 시작일 (yyyy-MM-dd) |
| endDate | String | X | 조회 종료일 (yyyy-MM-dd) |
| cursor | Long | X | 이전 응답의 `nextCursor` 값 |
| size | Int | X | 한 번에 불러올 개수 (기본: 20) |

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "녹음 목록 조회 성공",
  "data": {
    "content": [
      {
        "id": 5,
        "title": "운영체제 3주차",
        "duration": 2700,
        "analysisStatus": "COMPLETED",
        "createdAt": "2026-03-24T10:30:00"
      }
    ],
    "nextCursor": 4,
    "hasNext": true
  }
}
```

> `analysisStatus`: 분석 미요청 시 `null`, 이후 `PENDING` / `STT` / `SUMMARIZING` / `QUESTIONING` / `COMPLETED` / `FAILED`

### GET /recordings/{id}

녹음 상세 정보를 조회합니다. (FR-01)

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "녹음 상세 조회 성공",
  "data": {
    "id": 5,
    "title": "운영체제 3주차",
    "duration": 2700,
    "analysisStatus": "COMPLETED",
    "createdAt": "2026-03-24T10:30:00"
  }
}
```

### POST /recordings

새 녹음을 등록합니다. (FR-01)

- **Request Body**
```json
{
  "title": "운영체제 3주차",
  "duration": 2700
}
```

> `title` 생략 시 서버에서 `"3월 24일 녹음 (N)"` 형식으로 자동 생성합니다.

- **Response Body (201)**
```json
{
  "status": 201,
  "message": "녹음 등록 성공",
  "data": {
    "id": 1,
    "title": "3월 24일 녹음 (2)",
    "duration": 2700,
    "createdAt": "2026-03-24T10:30:00"
  }
}
```

### PATCH /recordings/{id}

녹음 제목을 수정합니다. (FR-01)

- **Request Body**
```json
{
  "title": "운영체제 3주차 (수정)"
}
```

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "녹음 제목 수정 성공",
  "data": {
    "id": 5,
    "title": "운영체제 3주차 (수정)"
  }
}
```

### DELETE /recordings/{id}

녹음 및 연관된 분석/질문 데이터를 삭제합니다. (FR-01)

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "녹음 삭제 성공"
}
```

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 409 | 분석 진행 중인 녹음 | "분석이 진행 중인 녹음은 삭제할 수 없습니다." |

## 파일 업로드 (Upload)

| Method | Endpoint | 설명 | 인증 |
|--------|----------|------|------|
| POST | `/recordings/{id}/upload-url` | Presigned URL 발급 | O |

### POST /recordings/{id}/upload-url

분석을 위한 음성 파일 업로드 URL을 발급합니다. 클라이언트는 이 URL로 Object Storage에 직접 업로드합니다.

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "업로드 URL 발급 성공",
  "data": {
    "uploadUrl": "https://object-storage.ncp.com/tmp/{recording_id}.m4a?signature=...",
    "expiredAt": "2026-03-24T11:00:00"
  }
}
```

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 409 | 이미 분석이 완료된 녹음 | "이미 분석이 완료된 녹음입니다." |

## 분석 (Analysis)

| Method | Endpoint | 설명 | 인증 |
|--------|----------|------|------|
| POST | `/recordings/{id}/analysis` | 분석 요청 | O |
| GET | `/recordings/{id}/analysis` | 분석 결과 조회 (요약 + 질문) | O |
| GET | `/recordings/{id}/analysis/transcript` | STT 변환 텍스트 조회 | O |
| GET | `/recordings/{id}/analysis/status` | 분석 상태 조회 | O |
| POST | `/recordings/{id}/analysis/retry` | 실패 단계부터 재시도 | O |

### POST /recordings/{id}/analysis

분석을 일괄 요청합니다. 사전에 Presigned URL로 음성 파일을 업로드한 상태여야 합니다. (FR-02)

- **Response Body (202)**
```json
{
  "status": 202,
  "message": "분석 요청 성공",
  "data": {
    "status": "PENDING"
  }
}
```

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 409 | 이미 분석이 진행 중 | "이미 분석이 진행 중입니다." |
| 409 | 이미 분석이 완료된 녹음 | "이미 분석이 완료된 녹음입니다." |
| 429 | 동시 분석 1건 초과 | "이미 분석이 진행 중입니다. 완료 후 다시 요청해주세요." |

### GET /recordings/{id}/analysis

분석 결과(요약본 + 질문)를 조회합니다. (FR-02)

- **Response Body (200) — COMPLETED**
```json
{
  "status": 200,
  "message": "분석 결과 조회 성공",
  "data": {
    "status": "COMPLETED",
    "summary": {
      "keyConcepts": [
        { "term": "프로세스 스케줄링", "definition": "CPU를 사용하려는 여러 프로세스 중 할당 대상을 결정하는 메커니즘" }
      ],
      "sections": [
        {
          "title": "주요 분류",
          "items": ["선점형: 실행 중인 프로세스를 강제 중단하고 CPU 재할당 가능", "비선점형: 프로세스가 자발적으로 CPU를 반납할 때까지 대기"]
        }
      ],
      "keywords": ["프로세스 스케줄링", "선점형", "비선점형"]
    },
    "questions": [
      { "content": "SJF 스케줄링의 기아 현상을 해결하기 위한 방법은?", "orderNum": 1 }
    ],
    "createdAt": "2026-03-24T10:35:00",
    "completedAt": "2026-03-24T10:37:00"
  }
}
```

> `status`가 `FAILED`일 때는 `failedStep`, `errorMessage`가 추가로 포함됩니다.

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 404 | 분석 미요청 | "분석 결과가 존재하지 않습니다." |

### GET /recordings/{id}/analysis/transcript

STT 변환 텍스트를 조회합니다. (FR-02)

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "변환 텍스트 조회 성공",
  "data": {
    "transcript": "오늘은 프로세스 스케줄링에 대해 알아보겠습니다. 스케줄링이란..."
  }
}
```

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 404 | STT 미완료 | "변환 텍스트가 존재하지 않습니다." |

### GET /recordings/{id}/analysis/status

분석 진행 상태를 폴링으로 확인합니다. (FR-02)

```json
{
  "status": 200,
  "message": "분석 상태 조회 성공",
  "data": {
    "analysisStatus": "SUMMARIZING",
    "currentStep": 2,
    "totalSteps": 3,
    "steps": [
      { "name": "STT", "status": "COMPLETED" },
      { "name": "SUMMARIZING", "status": "IN_PROGRESS" },
      { "name": "QUESTIONING", "status": "PENDING" }
    ]
  }
}
```

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 404 | 분석 미요청 | "분석이 요청되지 않은 녹음입니다." |

### POST /recordings/{id}/analysis/retry

실패 단계부터 재시도합니다. (FR-02)

- STT 실패: 클라우드 임시 파일로 재시도 (24시간 이내)
- 요약/질문 생성 실패: DB 저장 데이터로 재시도 (시간 제한 없음)

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 400 | 실패 상태가 아님 | "실패 상태인 분석만 재시도할 수 있습니다." |
| 410 | 임시 파일 만료 | "임시 파일이 만료되었습니다. 분석을 새로 요청해주세요." |

## 질문 (Question)

### POST /recordings/{id}/analysis/questions/regenerate

기존 요약본을 기반으로 심화 질문 5개를 재생성합니다. 기존 질문은 새 질문으로 대체됩니다. (FR-02)

- **Response Body (200)**
```json
{
  "status": 200,
  "message": "질문 재생성 성공",
  "data": {
    "questions": [
      { "content": "선점형 스케줄링에서 문맥 전환 비용을 최소화하는 방법은?", "orderNum": 1 },
      { "content": "기아 현상이 발생하는 근본적인 원인은?", "orderNum": 2 },
      { "content": "라운드 로빈 스케줄링의 타임 퀀텀 크기가 성능에 미치는 영향은?", "orderNum": 3 },
      { "content": "다단계 피드백 큐가 단일 큐 방식보다 유리한 상황은?", "orderNum": 4 },
      { "content": "실시간 시스템에서 데드라인 미스를 방지하기 위한 스케줄링 전략은?", "orderNum": 5 }
    ]
  }
}
```

| 상태 코드 | 상황 | 응답 메시지 |
|----------|------|-----------|
| 400 | 분석 미완료 | "분석이 완료된 녹음만 질문 재생성이 가능합니다." |
